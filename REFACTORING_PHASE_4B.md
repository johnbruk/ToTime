# TOTIME — Refactoring Phase 4B

## Obiettivo

Preparare l'introduzione di piccoli componenti riutilizzabili.

## Componenti candidati

- badge
- button
- card
- field
- empty state

## Regole

- Non cambiare la UI visibile nella prima PR operativa.
- Non modificare il database.
- Non modificare il service worker.
- Collegare i componenti una vista alla volta.

## Prossima fase

Creare i primi helper UI e usarli solo in una vista semplice.
